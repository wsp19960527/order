工单管理微信小程序（离线优先）

简介
- 面向小团队的工单管理，所有数据本地存储，离线可用，删除小程序即移除所有数据
- 主题色天蓝色，简约风格，移动端友好，兼容 iOS 安全区

核心功能
- 工单增删改查，状态流转：待处理、处理中、已完成、延期
- 列表筛选与分页：按状态筛选；支持“临近截止”提醒（截止前 2 天）与“延期”自动判定
- 详情页操作：根据状态显示可用操作；底部固定操作栏不遮挡内容

数据存储设计
- 单键数组存储：所有工单保存在 `app:tickets`（JSON 数组）中
- 工单主键：`id` 为 12 位随机字符串（`a-z0-9`），避免冲突
- 状态规则：
  - 旧状态“已解决/已关闭”统一映射为“已完成”
  - 有 `dueDate` 且已过期、非“已完成”则标记为“延期”

数据模型示例
```
{
  "id": "a1b2c3d4e5f6",
  "title": "修复登录异常",
  "description": "iOS 设备偶发错误码 401",
  "priority": "中",
  "status": "待处理",
  "assignee": "Alice",
  "dueDate": "2025-12-10",
  "createdAt": 1733318400000,
  "updatedAt": 1733404800000
}
```

页面与路由
- 列表页：`pages/list/index`，状态 Chips、统计仪表盘（总数/待完成/本月新增）、卡片信息展示（描述、优先级、负责人、截止日期）
- 详情页：`pages/detail/index`，标题、状态徽章、描述、字段信息；底部操作栏（等宽按钮）
- 编辑页：`pages/edit/index`，表单控件统一尺寸；日期选择器整块区域可点击；底部保存按钮全宽





注意事项
- 所有数据保存在本地 `storage`，清理缓存/卸载小程序将移除数据
- 当前不包含附件落地，仅保存元数据；后续可接入云开发/后端进行同步
